# 34-夜间灯光占位效果实现计划

## 目标

本节实现夜间灯光的第一版可见效果：夜晚时，地图里的路灯或窗户光在
对应位置显示暖色光圈；白天自动熄灭。

这一版不实现 C++ 的完整多 pass 光照管线。Zig 先落地光源数据、时间显隐
规则和可见占位绘制，保持实现直接、可测试、可逐步替换。

## C++ 实现梳理

C++ 这一节主要由四部分组成：

- `PointLightComponent`、`SpotLightComponent`、`EmissiveRectComponent`、
  `EmissiveSpriteComponent` 挂在实体上。
- `TimeOfDayLightSystem` 根据整点事件和视觉暗时段，添加或移除
  `LightDisabledTag`。
- `LightSystem` 收集没有禁用的光源组件，提交给 Renderer。
- Renderer 使用 `LightingPass`、`EmissivePass`、`BloomPass`、
  `CompositePass` 合成最终画面。

C++ 的关键取舍是把游戏层和渲染层分开：游戏层只关心光源实体和显隐规则，
渲染层只关心当前帧有哪些光源需要提交。

## Zig 实现取舍

当前 Zig 引擎只有单 pass batch 渲染。如果现在照搬 C++ 的
`LightingPass + CompositePass + BloomPass`，会把本节扩大成渲染器重构。

Zig 第一版采用更小的实现：

- 光源仍然作为 ECS 组件存在。
- 显隐规则仍然由时间驱动。
- 绘制先使用现有 `circle.png` 画半透明暖色光圈。
- 不新增 shader，不新增 framebuffer，不修改 engine renderer。

这样后续如果要做真实光照，可以复用本节的光源组件和地图加载逻辑，只替换
绘制提交部分。

## 34.1 光源组件和暗时段判断

目标：先建立光源数据，不画东西。

修改文件：

- `farm/component.zig`
- `farm/system/light.zig`

新增命名：

```zig
component.light.Point
component.light.Spot
component.light.Disabled
component.light.NightOnly
component.light.DayOnly
```

实现方式：

- `Point` 保存 `radius`、`offset`、`color`、`intensity`。
- `Spot` 第一版只保存 `radius`、`direction`、`color`、`intensity`。
- `Disabled` 表示关灯但实体仍然存在。
- `isDark(hour, minute)` 使用 C++ 配置语义：`18:00 <= time < 6:00`。

C++ 对比与取舍：

- C++ 分 engine component 和 game tag。
- Zig 直接放在 `component.light`，避免增加 engine/game 两层抽象。
- C++ 从 JSON 加载 `emissive_visibility`。
- Zig 第一版用常量，后续需要调参再放 ZON。

验证方式：

- 测试 `17:59` 不暗。
- 测试 `18:00` 暗。
- 测试 `05:59` 暗。
- 测试 `06:00` 不暗。
- 运行 `zig build test`。

本步不做：

- 不解析地图对象。
- 不绘制光圈。
- 不做玩家灯开关。

## 34.2 加载 Tiled 光源对象

目标：让 `farm/zon/town.zon` 里的 `type = "light"` 对象进入 ECS。

修改文件：

- `farm/map.zig`
- `farm/factory.zig`
- `farm/component.zig`

新增命名：

```zig
factory.spawnPointLight()
factory.spawnSpotLight()
```

实现方式：

- `object.name == "point"` 时读取 `radius` 属性，生成 `Point`。
- `night_only` 和 `day_only` 转成对应 tag。
- 加载时立即根据当前时间决定是否加 `Disabled`。
- `object.name == "spot"` 时，第一版先按默认参数生成 `Spot`，或者退化成
  点光占位。

C++ 对比与取舍：

- C++ 能从 Tiled class 里读取 `radius`、`inner_deg`、`outer_deg`、
  `direction_deg`。
- 当前 Zig 的 ZON 中 `spot` 只保留 `.class = "Spotlight"`，嵌套参数已经
  丢失。
- 第一版不修改 Tiled 转换器，避免把本节扩大到资源管线改造。

验证方式：

- 构造 `type = "light"` 的点光对象，确认生成 `Position + Point + Scoped +
  NightOnly`。
- 白天加载时应带 `Disabled`。
- 夜晚加载时不带 `Disabled`。
- 运行 `zig build test`。

本步不做：

- 不做真实锥形 spot。
- 不保留 Tiled class 嵌套属性。
- 不实现 emissive sprite 或 emissive rect。

## 34.3 光源显隐系统

目标：整点切换灯光显隐。

修改文件：

- `farm/system/light.zig`
- `farm/scene.zig`

新增命名：

```zig
system.light.update(world)
```

实现方式：

- 在 `system.time.update(world, delta)` 之后调用 `system.light.update(world)`。
- `light.update` 读取 `HourChanged` 事件。
- 有整点变化时遍历 `NightOnly` 和 `DayOnly` 光源。
- 根据 `isDark` 添加或移除 `Disabled`。

C++ 对比与取舍：

- C++ 用 dispatcher 订阅 `HourChangedEvent`。
- Zig 直接读取 `world` 里的事件队列，不增加订阅生命周期管理。
- 保留 C++ 的核心语义：只禁用光照，不隐藏实体。

验证方式：

- 测试 12 点时 night-only 禁用、day-only 启用。
- 测试 19 点时 night-only 启用、day-only 禁用。
- 运行 `zig build test`。

本步不做：

- 不每帧轮询所有光源。
- 不做亮度渐变。

## 34.4 绘制夜间光圈占位

目标：夜晚在路灯附近看到暖色光圈。

修改文件：

- `farm/system/light.zig`
- `farm/scene.zig`

新增命名：

```zig
system.light.init()
system.light.drawOverlay()
system.light.drawWorld(world)
```

实现方式：

- `drawOverlay()` 保留当前昼夜全屏 overlay。
- `drawWorld(world)` 遍历没有 `Disabled` 的 `Point` 光源。
- 使用现有 `circle.png` 画两层或三层半透明暖色圆。
- 外圈半径大、alpha 小。
- 内圈半径小、alpha 稍高。
- 场景统一控制相机模式：
  - 世界模式画地图、实体、前景、目标和世界光圈。
  - 窗口模式画昼夜 overlay、时钟、工具栏和对话框。

C++ 对比与取舍：

- C++ 使用 `LightingPass` 加法光照，再由 `CompositePass` 合成。
- Zig 第一版只是 alpha 光圈，不能真正照亮被 overlay 变暗的场景材质。
- 这个版本能快速验证地图光源、时间显隐和绘制位置。

验证方式：

- 测试启用的点光会写入 batch vertex。
- 测试禁用的点光不会写入 batch vertex。
- 运行 `zig build test`。
- 运行 `zig build`。
- 手动运行后把时间推进到夜晚，确认光圈可见。

本步不做：

- 不做 Bloom。
- 不做 emissive texture。
- 不做真实 additive blend。
- 不做遮挡和阴影。

## 34.5 玩家跟随灯开关

目标：补上 C++ `LightToggleSystem` 的精简版。

修改文件：

- `farm/system/light.zig`
- `farm/scene.zig`
- 可能少量修改 `farm/component.zig`

实现方式：

- 给玩家挂一个 `Point`，半径默认 `128`，偏移 `.xy(0, -10)`。
- 用按键切换 `wanted_on`。
- 室外只允许暗时段点亮。
- 当前还没有室内地图元数据，第一版只按暗时段控制。
- 不走事件系统，直接在 `light.update` 里处理输入。

C++ 对比与取舍：

- C++ 有 `ToggleLightRequest`、`LightToggleSystem` 和配置文件。
- Zig 第一版使用直接状态和常量，等输入和配置系统更完整后再抽出。

验证方式：

- 测试玩家灯白天禁用。
- 测试玩家灯夜晚允许。
- 手动运行按键，确认光圈跟随玩家。

本步不做：

- 不做室内地图分支。
- 不做独立配置文件。
- 不做输入 action 映射系统。

## 建议顺序

建议按下面顺序实现：

1. `34.1` 光源组件和暗时段判断。
2. `34.2` 加载 Tiled 光源对象。
3. `34.3` 光源显隐系统。
4. `34.4` 绘制夜间光圈占位。
5. `34.5` 玩家跟随灯开关。

这样每一步都能独立编译、独立测试，不把输入、地图、渲染和资源管线一次性
混在一起。
