# 实现原则

方案和实现基于第一原理，从最佳实践考虑，不要考虑修改范围和兼容性。
修改前最好先列计划，我确认后再修改，不然可能和我想的实现不一致。

1. 优先选用简单单词，我的母语不是英语，保持代码行长度不超过 80 个字符。
2. 方法、重要变量和代码应该有中文注释，命名遵循范围越大，名字越长。
3. 不要抽象、兜底和防御，需要简单和直接。提前暴露错误，避免运行正常，逻辑错误。
4. 不要无脑提取小方法，方法参数不宜过多，超过 3 个就考虑拆分或使用结构体。
5. 出现的数值，字符串，直接写到使用的地方，禁止提取常量，避免上下文查找。
6. 优先使用 Zig 风格的简单直接实现，避免照搬 C++ 的强封装、继承和复杂框架。
