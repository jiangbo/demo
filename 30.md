# 30-物品使用与农场循环 实现计划

## 目标

对照 C++ 教程 `30-物品使用与农场循环.md`，在 Zig 版 Tiny Farm 中
完成并校验农场玩法主线：

```text
快捷栏选择工具/种子 -> 点击目标格 -> 改变世界状态
-> 作物跨天生长 -> 镰刀收获成熟作物并生成掉落 -> 拾取入包
-> 右键使用作物获得种子
```

当前 Zig 代码已经有一部分实现，主要在：

- `farm/system/control.zig`
- `farm/map/land.zig`
- `farm/map.zig`
- `farm/system/pickup.zig`
- `farm/inventory.zig`

本计划先补齐缺口，再更新 `plan.md` 中与第 30 讲相关的状态。

## 1. 现状校验（已完成）

已运行 `zig build` 和 `zig build test`，当前工程可以编译，现有测试全部通过。
第 30 讲已有链路具备可继续迭代的干净基线。

## 2. 补地图语义检查（已完成）

已把 Tiled 地图语义检查放在 `map.spatial`，把农场地块状态检查放在
`map.land`，并补齐 `canHoe`、`canPlant`、`canWater` 的规则和测试。

C++ 对照与取舍：

- C++ 使用更完整的空间索引和 tile entity。
- Zig 先保留 `spatial + land.Tile` 的直接结构，后续资源节点再扩展。

## 3. 实现动画关键帧结算（已完成）

已完成关键帧结算主链路：

```text
点击 -> 写 WantUse + Busy + Actor.action
-> 动画进入 extend 关键帧 -> 挂 UseFrame
-> farm 系统结算 Player 的 WantUse
-> 清理 UseFrame + WantUse
-> 动画结束解除 Busy
```

`control.zig` 只写意图，`animation.zig` 只按 `Frame.extend` 标记生效帧，
`farm/system/farm.zig` 负责真正修改农场状态。

C++ 对照与取舍：

- C++ 通过动画事件系统转发到 `FarmSystem`。
- Zig 直接用 `UseFrame` 组件表达关键帧，只处理当前玩家，保持实现简单。

## 4. 补种植消耗边界（已完成）

种植结算已收口到 `farm.useSeed()`：先确认目标格可种植，再通过
`inventory.use(seed, 1)` 扣种子，成功后创建作物并播放种植音效。

已补测试覆盖：

- 最后一颗种子种下后，快捷栏不再返回可用物品。
- 没有种子时不会种植。
- 无耕地或已有作物时不会扣种子。
- `inventory.use()` 不接受 `count == 0` 的调用。

C++ 对照与取舍：

- C++ 通过 `RemoveItemRequest` 指定 inventory slot，同步扣减。
- Zig 不做事件链，继续让 `inventory.zig` 作为库存单一状态模块。
- Zig 当前按物品类型扣种子，不把槽位事务下沉到引擎。

## 5. 补镰刀收获与掉落路径（已完成）

已补齐工具物品配置，并把成熟作物收获限定到镰刀：

```text
成熟作物 -> 镰刀收获 -> spawnPickup()
-> 玩家碰撞 pickup -> inventory.add()
```

已补测试覆盖：

- 镰刀收获会清理 `tile.object`、销毁作物、生成 pickup、播放 `harvest`。
- 锄头、水壶、种子不会收获成熟作物。
- 背包满时 pickup 不消失。
- 背包只有部分空间时，pickup 只减少成功拾取的数量。

C++ 对照与取舍：

- C++ 教程主线里收获直接进背包，课后练习才改为 pickup。
- Zig 保留 pickup 路径，入包仍由 `pickup.update()` 直接调用 `inventory.add()`。

## 6. 完善作物日结算测试（已完成）

作物日结算已由 `map.update()` 消费 `DayChanged` 事件处理，离线地图恢复
复用同一套 `advanceCropOneDay()` 规则。

已补测试覆盖：

- 离线地图按天数差推进作物。
- 湿地离线跨天只加速一天。
- 当前地图收到 `DayChanged` 后推进作物并清干湿地。
- 成熟作物不再推进。
- 当前地图和离线地图推进规则一致。
- 推进阶段后会刷新贴图，并把非 seed 阶段切到 `.actor` 渲染层。

C++ 对照与取舍：

- C++ 使用独立 `CropSystem` 订阅 `DayChangedEvent`。
- Zig 直接在 `map.update()` 中消费 `DayChanged` 并遍历 `land.tiles`，
  当前地图状态和存档状态共用同一套推进函数。

## 7. 右键物品使用与计划同步

当前 `farm/inventory.zig` 已有：

- `updateUseItem()`
- `useItem(index)`
- 右键使用成功后通过 `context.notice.show(.item, "获得 {s} x{d}", ...)`
  显示物品提示。
- 右键背包槽使用物品测试
- 右键快捷栏使用绑定物品测试
- 空间不足时不修改背包测试

Zig 计划：

- 跑测试确认右键 UseItem 路径有效。
- 补或确认测试：成功使用作物后，`.item` 提示频道显示“获得 Xxx x3”。
- `plan.md` 中过期的右键 `UseItem` 待办不再保留。

C++ 对照与取舍：

- C++ 使用 `UseItemRequest` 和 `ItemUseSystem`。
- Zig 直接在 `inventory.zig` 内完成 simulate-then-execute。
- 这保留了原子性，但不新增独立系统。
- C++ 路径 D 的提示气泡在 Zig 中复用现有 `notice .item` 频道，
  由 `drawItemNotice()` 画在快捷栏上方。

## 8. 资源节点暂不纳入本步主线

C++ 第 30 讲包含：

- `Axe`
- `Pickaxe`
- 树和石头耐久
- 资源节点掉落
- 空间索引清理

Zig 当前 `plan.md` 已记录资源节点仍是后续功能：

- `ROCK` 只是 `land.Object.kind` 能表达。
- 地图加载尚未把石头写入 `tile.object`。
- 工具系统还没有敲石头、销毁实体和清理格子的逻辑。

Zig 计划：

- 本步不实现完整资源节点。
- 保留为后续独立任务。
- 第 30 讲只闭合农场主线：地块、作物、掉落、拾取、物品使用。

C++ 对照与取舍：

- C++ 同节覆盖更多系统，完整度更高。
- Zig 本步避免扩大范围，不引入资源节点框架。
- 后续做资源节点时再补 `rock`/`tree` 状态、命中次数和清理函数。
